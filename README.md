# Typing Vocab / 타자 연습 도구

打字背单词 demo，灵感来自 [metrotyping.kr](https://metrotyping.kr)（韩国地铁打字游戏）的交互形式：
当前字符消失只留下你实际打出的内容，逐部件（초성/중성/종성）高亮对错，配合小提示和进度统计。

## 核心架构：不依赖浏览器输入法（IME）

这版和早期版本最大的区别：**韩语输入不再通过浏览器原生的 Hangul 输入法组字**
（不监听 `compositionstart`/`compositionend`/`<input>` 的 `value`），而是直接监听
物理键盘的 `keydown` 事件（`event.code`，即物理按键位置），按标准**두벌식（2-set）**
键盘映射表自己实现了一套完整的韩语组字引擎（`makeKoreanEngine`）。

这么做的原因：早期版本反复出现"打第二个字时，内容跑到第一个字位置"的乱序问题，
根源是浏览器的原生 Hangul 组字状态和网页脚本对 `<input>` 内容的操作互相脱节
（尤其是换词时清空输入框、光标位置被浏览器不可预期地重置等）。改成自己实现组字
引擎后，整个过程完全在 JS 里确定性地跑（`cho`/`jung`/`jong` 状态机 + `activeIndex`
显式维护），不再依赖、也不会被浏览器/操作系统的输入法行为影响，从根上避开了这类问题。

**这意味着：使用时不需要把系统切换成韩语输入法**，用英文/QWERTY 物理键盘直接打字
即可，页面会按두벌식规则把物理按键翻译成对应字母（比如 `d`→ㅇ, `u`→ㅕ, `s`→ㄴ 打出"연"）。

## 特点

- **韩语模式**：按음절（音节）显示，每个字拆成 초성/중성/종성，逐部件比对：
  - 灰色（浅）：还没轮到的字
  - 深灰色：轮到的字 / 正在打且目前正确
  - 蓝色：整个字打完且完全正确
  - 红色：这个字有部分打错了（不影响继续打下一个字），上方会浮现一个小提示
    显示正确写法，直到你用 Backspace 改对为止
  - **横杠数按实际物理按键数算**，不是简单按"有没有收音"算：复合元音
    （ㅘㅙㅚㅝㅞㅟㅢ）和复合收音（ㄳㄵㄶㄺㄻㄼㄽㄾㄿㅀㅄ）在두벌식键盘上
    都需要按两次键才能拼出来，所以各占 2 条杠。比如"화"虽然没有收音，
    也是 3 条杠（ㅎ + ㅗ + ㅏ 三次按键），不是 2 条。
  - 收音后面如果又打了元音，收音会正确地挪到下一个字当声母（比如打
    "닭"再打"ㅣ"会变成"달"+"기"）。
- **英语模式**：逐字母实时比对，每个字母位置独立判断对错（互不影响），
  对了变绿、错了变红。
- `Backspace` 逐个部件/字母删除；打对整个单词后按 `Enter` 进入下一个词，
  或点"다음 단어"跳过。
- 可选"딕테이션 모드"（听写模式）：勾选后英语模式不再预先显示单词提示。

## 使用方法

直接用浏览器打开 `index.html` 即可（无需安装依赖、无需构建）。**不需要切换系统
输入法**——保持英文/默认键盘布局，页面会自己按두벌식规则解释你的按键。

## 词表

`index.html` 里的 `KO_WORDS`（`{word, roman}`）和 `EN_WORDS`（纯字符串数组）
两个数组内置了少量示例词，换成自己的词表直接替换这两个数组即可。

> 注意：这版的词表只有罗马音注音，没有中文释义字段——如果要恢复"背单词"
> 场景下的释义显示，需要给 `KO_WORDS`/`EN_WORDS` 加一个释义字段并在
> `render()` 里显示出来。

## 已知局限

- 目前只支持标准두벌식键盘映射，没有支持其他韩语输入方案（如세벌식）。
- 没有间隔重复（spaced repetition）逻辑，线性走完词表。
- 没有中文释义展示（见上）。
