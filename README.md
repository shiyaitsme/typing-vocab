# Typing Vocab / 타자 연습 도구

打字背单词 demo，灵感来自 [metrotyping.kr](https://metrotyping.kr)（韩国地铁打字游戏）的交互形式：
当前字符消失只留下你实际打出的内容，逐部件（초성/중성/종성）高亮对错，配合小提示和进度统计。

## 核心架构：不依赖浏览器输入法（IME）

这版和早期版本最大的区别：**韩语输入不再通过浏览器原生的 Hangul 输入法组字**
（不监听 `compositionstart`/`compositionend`/`<input>` 的 `value`），而是直接监听
物理键盘的 `keydown` 事件（`event.code`，即物理按键位置），按标准**두벌식（2-set）**
键盘映射表自己实现了一套完整的韩语组字引擎（`makeKoreanEngine`）。

这么做的原因：早期版本反复出现"打第二个字时，内容跑到第一个字位置"的乱序问题，
根源是浏览器的原生 Hangul 组字状态和网页脚本对 `<input>` 内容的操作互相脱节
（尤其是换词时清空输入框、光标位置被浏览器不可预期地重置等）。改成自己实现组字
引擎后，整个过程完全在 JS 里确定性地跑（`cho`/`jung`/`jong` 状态机 + `activeIndex`
显式维护），不再依赖、也不会被浏览器/操作系统的输入法行为影响，从根上避开了这类问题。

**这意味着：使用时不需要把系统切换成韩语输入法**，用英文/QWERTY 物理键盘直接打字
即可，页面会按두벌식规则把物理按键翻译成对应字母（比如 `d`→ㅇ, `u`→ㅕ, `s`→ㄴ 打出"연"）。

## 特点

- **韩语模式**：按음절（音节）显示，每个字拆成 초성/중성/종성，逐部件比对：
  - 灰色（浅）：还没轮到的字
  - 深灰色：轮到的字 / 正在打且目前正确
  - 蓝色：整个字打完且完全正确
  - 红色：这个字有部分打错了（不影响继续打下一个字），上方会浮现一个小提示
    显示正确写法，直到你用 Backspace 改对为止
  - **横杠数按实际物理按键数算**，不是简单按"有没有收音"算：复合元音
    （ㅘㅙㅚㅝㅞㅟㅢ）和复合收音（ㄳㄵㄶㄺㄻㄼㄽㄾㄿㅀㅄ）在두벌식键盘上
    都需要按两次键才能拼出来，所以各占 2 条杠。比如"화"虽然没有收音，
    也是 3 条杠（ㅎ + ㅗ + ㅏ 三次按键），不是 2 条。
  - 收音后面如果又打了元音，收音会正确地挪到下一个字当声母（比如打
    "닭"再打"ㅣ"会变成"달"+"기"）。
- **英语模式**：逐字母实时比对，每个字母位置独立判断对错（互不影响），
  对了变绿、错了变红。
- `Backspace` 逐个部件/字母删除；打对整个单词后按 `Enter` 进入下一个词，
  或点"다음 단어"跳过。
- 可选"딕테이션 모드"（听写模式）：勾选后英语模式不再预先显示单词提示。
- **间隔重复（복습 모드）**：每个词有一个 0-5 级的"熟练度"（Leitner box）。一次
  不出错就打对，等级 +1，下次复习间隔拉长（1/3/7/14/30 天）；打错了等级 -1，
  很快又会到期。勾选"🔁 복습 모드"后，只在当前到期的词里走（跳过还没到期的），
  某个模式下今天没有到期的词会弹提示并自动关闭复习模式。复习进度按等级/到期
  日存在词条本身上，跟收藏、笔记一样存云端。
- **正确率 / 打字速度**：进度条那行现在会显示本次打开页面以来的"정확도"
  （某个词从头到尾一次都没打错就算"clean"，clean 词数 / 完成词数）和
  "CPM"（已完成词的物理按键数 / 经过的分钟数），刷新页面会重置。
- **错词本（오답만）**：只要某个词在这次作答过程中出现过打错（哪怕后来改对
  了），就会给它的"오답 횟수"+1，单词列表里会显示 `❌N`，抽屉里勾选
  "❌ 오답만"就能只看这些词——不用手动收藏，打错自动进错词本。

## 使用方法

直接用浏览器打开 `index.html` 即可（无需安装依赖、无需构建）。**不需要切换系统
输入法**——保持英文/默认键盘布局，页面会自己按두벌식规则解释你的按键。

## 词表

`index.html` 里的 `KO_WORDS`（`{word, roman}`）和 `EN_WORDS`（纯字符串数组）
两个数组内置了少量示例词，换成自己的词表直接替换这两个数组即可。

> 注意：这版的词表只有罗马音注音，没有中文释义字段——如果要恢复"背单词"
> 场景下的释义显示，需要给 `KO_WORDS`/`EN_WORDS` 加一个释义字段并在
> `render()` 里显示出来。

## 已知局限

- 目前只支持标准두벌식键盘映射，没有支持其他韩语输入方案（如세벌식）。
- 没有中文释义展示（见上）。
- 正确率/CPM 只是当次打开页面这段时间的粗略统计，不做历史趋势记录。

## 云端存储部署（Cloudflare Pages + D1）

单词、收藏、笔记、每日学习历史现在存在 Cloudflare D1 数据库里，不再依赖浏览器
本地存储——换设备、换浏览器、清缓存都不会丢。不设密码，靠首次访问时填的名字
区分各自的数据（同名 = 同一份数据，可用来在多设备/多人间共享同一账号）。

代码（`functions/api/words.js`、`functions/api/history.js`、`schema.sql`、
`wrangler.toml`）已经写好并提交在仓库里，但**实际创建数据库 + 部署这一步需要
你自己的 Cloudflare 账号**，下面是完整步骤：

```bash
# 1. 登录 Cloudflare（会打开浏览器授权）
npx wrangler login

# 2. 创建 D1 数据库
npx wrangler d1 create typing-vocab-db
# 输出里会有一行 database_id = "xxxxxxxx-xxxx-...", 复制它

# 3. 把 database_id 填进 wrangler.toml
#    打开 wrangler.toml，把 REPLACE_WITH_YOUR_D1_DATABASE_ID 换成上一步拿到的 id

# 4. 把表结构建到远程数据库
npx wrangler d1 execute typing-vocab-db --remote --file=./schema.sql

# 5. 部署（第一次会让你确认创建 Pages 项目）
npx wrangler pages deploy .
```

部署完成后 wrangler 会给一个 `*.pages.dev` 的网址，直接打开就能用。以后每次
改完 `index.html` 或 `functions/` 下的代码，重新跑一次 `npx wrangler pages deploy .`
（或接入 Git 自动部署）即可更新线上版本。

**如果之前已经建过库**（`words` 表里还没有 `box`/`due_at`/`wrong_count` 这几列，
即间隔重复/错词本相关字段），重新跑一遍下面这条把新字段加上去即可，不会丢数据：

```bash
npx wrangler d1 execute typing-vocab-db --remote --file=./schema.sql
```

**本地开发调试**（不需要真的部署，用本地模拟的数据库测试后端逻辑）：

```bash
npx wrangler d1 execute typing-vocab-db --local --file=./schema.sql
npx wrangler pages dev .
```

如果 `/api/words`、`/api/history` 接口访问不到（比如还没部署好、或本地直接用
`python -m http.server` 这种纯静态服务器打开），页面会自动降级：用内置的默认
词表跑起来，并在进度条下方显示一行提示，说明这次的改动不会被保存——不会白屏
或报错卡住。
